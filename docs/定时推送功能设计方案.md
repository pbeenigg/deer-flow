# DeerFlow 定时推送功能实现方案

> 版本：v1.0  
> 日期：2026-04-16  
> 目标：为 DeerFlow 增加定时任务能力，支持每日自动推送热点信息

---

## 一、方案概述

### 1.1 核心目标

让 Agent 具备**主动触达用户**的能力，实现：
- 用户配置定时任务（如"每天9点推送科技热点"）
- 系统自动在指定时间执行任务
- 结果主动推送给用户（邮件/微信/Telegram等）

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **最小侵入** | 尽量复用现有 Agent 能力，核心逻辑不改 |
| **渐进式** | 分阶段实现，每阶段可独立上线 |
| **可扩展** | 支持多种推送渠道，易于新增 |
| **高可靠** | 任务不丢失，失败可重试 |

### 2.2 数据流

```
1. 用户配置任务
   用户 → Web/API → 任务管理服务 → 存储到数据库

2. 定时触发
   调度器 → 扫描到期任务 → 发送到消息队列

3. 执行任务
   Agent Worker ← 拉取任务 ← 消息队列
   Agent Worker → 调用现有 Agent 能力 → 生成内容

4. 推送结果
   Agent Worker → 推送服务 → 邮件/微信/Telegram → 用户
   Agent Worker → 更新任务状态 → 数据库
```

### 3.2 任务调度器（Scheduler）

**职责：**
- 扫描到期的任务
- 计算下次执行时间
- 将任务发送到消息队列

**技术选型：**

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **APScheduler** | Python原生，简单易用 | 单进程，不适合分布式 | 小规模、单机部署 |
| **Celery Beat** | 成熟稳定，分布式 | 依赖 Redis/RabbitMQ | 中大规模、分布式 |
| **云服务调度** | 无需维护，高可用 | 依赖云平台 | 云原生部署 |
| **自建调度服务** | 完全可控 | 开发成本高 | 定制化需求 |

**推荐方案：APScheduler + Redis 分布式锁**（兼顾简单与可靠）

```python
# scheduler.py
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger
import redis

class TaskScheduler:
    def __init__(self):
        self.scheduler = AsyncIOScheduler()
        self.redis = redis.Redis(host='localhost', port=6379)
        self.queue = TaskQueue()
    
    async def start(self):
        """启动调度器"""
        # 每分钟扫描一次到期任务
        self.scheduler.add_job(
            self.scan_due_tasks,
            'interval',
            minutes=1,
            id='scan_due_tasks'
        )
        self.scheduler.start()
    
    async def scan_due_tasks(self):
        """扫描到期任务"""
        # 分布式锁，防止多实例重复执行
        lock = self.redis.lock('scheduler_lock', timeout=30)
        
        if lock.acquire(blocking=False):
            try:
                # 查询到期任务
                tasks = await self.get_due_tasks()
                
                for task in tasks:
                    # 发送到消息队列
                    await self.queue.enqueue(task)
                    
                    # 更新下次执行时间
                    await self.update_next_run_time(task)
            finally:
                lock.release()
    
    async def get_due_tasks(self):
        """获取到期的任务"""

            SELECT * FROM scheduled_tasks
```

### 3.3 消息队列（Queue）

**职责：**
- 解耦调度器和执行器
- 任务持久化，防止丢失
- 支持重试和优先级

**技术选型：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Redis Queue (RQ)** | 简单，Python原生 | 功能较少 |
| **Celery** | 功能强大，生态完善 | 较重 |
| **RabbitMQ** | 企业级，高可靠 | 运维复杂 |

**推荐方案：Redis Queue（RQ）** 或 **Celery**

```python
# queue.py
from rq import Queue
from redis import Redis

redis_conn = Redis()
task_queue = Queue('scheduled_tasks', connection=redis_conn)

async def enqueue_task(task):
    """将任务加入队列"""
    job = task_queue.enqueue(
        'worker.execute_task',
        task_id=task.id,
        retry=Retry(max=3, interval=[60, 300, 900])  # 失败重试
    )
    return job.id
```

### 3.4 Agent Worker

**职责：**
- 从队列获取任务
- 调用现有 Agent 能力执行任务
- 调用推送服务发送结果
- 更新任务执行状态

**关键设计：复用现有 Agent**

```python
# worker.py
from deerflow.agent import Agent
from deerflow.memory import MemoryManager
from services.notify import NotifyService

async def execute_task(task_id: str):
    """执行定时任务"""
    # 1. 获取任务配置
    task = await get_task(task_id)
    execution = await create_execution(task_id)
    
    try:
        # 2. 初始化 Agent（复用现有能力）
        agent = Agent(
            model=task.model_config,
            memory=MemoryManager(thread_id=task.thread_id)
        )
        
        # 3. 执行任务
        result = await agent.run(task.task_prompt)
        
        # 4. 推送结果
        notify_service = NotifyService()
        await notify_service.send(
            channels=task.notify_channels,
            content=result.content,
            config=task.notify_config
        )
        
        # 5. 更新状态
        await update_execution(execution.id, status='success', result=result.content)
        
    except Exception as e:
        await update_execution(execution.id, status='failed', error=str(e))
        raise  # 触发重试
```

### 3.5 推送服务（Notify Service）

**职责：**
- 统一的推送接口
- 支持多种推送渠道
- 推送状态追踪

**渠道适配器模式：**

```python
# notify.py
from abc import ABC, abstractmethod
from typing import List, Dict

class NotifyChannel(ABC):
    """推送渠道基类"""
    
    @abstractmethod
    async def send(self, content: str, config: dict) -> dict:
        """发送消息，返回发送状态"""
        pass

class EmailChannel(NotifyChannel):
    """邮件推送"""
    
    async def send(self, content: str, config: dict) -> dict:
        # 使用 SMTP 或第三方服务（如 SendGrid）
        import smtplib
        from email.mime.text import MIMEText
        
        msg = MIMEText(content, 'html')
        msg['Subject'] = config.get('subject', 'DeerFlow 定时推送')
        msg['From'] = config['from_email']
        msg['To'] = config['to_email']
        
        # 发送邮件
        # ...
        return {"status": "success", "message_id": "xxx"}

class WechatChannel(NotifyChannel):
    """企业微信机器人"""
    
    async def send(self, content: str, config: dict) -> dict:
        webhook_url = config['webhook_url']
        
        async with aiohttp.ClientSession() as session:
            payload = {
                "msgtype": "markdown",
                "markdown": {"content": content}
            }
            async with session.post(webhook_url, json=payload) as resp:
                return await resp.json()

class TelegramChannel(NotifyChannel):
    """Telegram Bot"""
    
    async def send(self, content: str, config: dict) -> dict:
        bot_token = config['bot_token']
        chat_id = config['chat_id']
        
        url = f"https://api.telegram.org/bot{bot_token}/sendMessage"
        
        async with aiohttp.ClientSession() as session:
            payload = {
                "chat_id": chat_id,
                "text": content,
                "parse_mode": "Markdown"
            }
            async with session.post(url, json=payload) as resp:
                return await resp.json()

class DingTalkChannel(NotifyChannel):
    """钉钉机器人"""
    
    async def send(self, content: str, config: dict) -> dict:
        webhook_url = config['webhook_url']
        
        async with aiohttp.ClientSession() as session:
            payload = {
                "msgtype": "markdown",
                "markdown": {"title": "DeerFlow 推送", "text": content}
            }
            async with session.post(webhook_url, json=payload) as resp:
                return await resp.json()

class NotifyService:
    """推送服务"""
    
    channels = {
        'email': EmailChannel(),
        'wechat': WechatChannel(),
        'telegram': TelegramChannel(),
        'dingtalk': DingTalkChannel()
    }
    
    async def send(self, channels: List[str], content: str, config: dict) -> dict:
        """发送到多个渠道"""
        results = {}
        
        for channel_name in channels:
            channel = self.channels.get(channel_name)
            if channel:
                try:
                    result = await channel.send(content, config.get(channel_name, {}))
                    results[channel_name] = result
                except Exception as e:
                    results[channel_name] = {"status": "failed", "error": str(e)}
        
        return results
```

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 任务执行失败 | 用户收不到推送 | 重试机制 + 失败告警 |
| 调度器宕机 | 所有任务停止 | 多实例 + 健康检查 |
| 消息队列积压 | 推送延迟 | 监控告警 + 扩容 Worker |
| 推送渠道限流 | 推送失败 | 限流控制 + 降级策略 |
| 时区处理错误 | 推送时间错误 | 统一使用 UTC 存储 |
